# 部署军规

## 核心原则

所有部署相关操作必须通过 deploy.sh 完成，禁止手动执行零散命令。

## 规则

1. 任何涉及服务启动、停止、重启、迁移的操作，必须集成到 deploy.sh 中
2. 不允许让用户手动执行 docker compose run/exec 来完成部署步骤
3. 新增服务、配置变更、环境变量调整都必须更新到 deploy.sh 和 docker-compose.yml
4. deploy.sh 必须幂等可重复执行，已完成的步骤自动跳过
5. deploy.sh 遇到错误时给出明确提示，不能静默失败
6. 所有踩过的坑必须在 deploy.sh 中自动处理，不能依赖人工记忆
7. 优先使用 PostHog 官方提供的脚本和能力，不在未调研的情况下自己造轮子
8. 遇到不确定的问题，先查官方文档和源码，不要猜测

## 修改 deploy.sh 时的检查清单

- 新增步骤是否有跳过检测（幂等性）
- 管道命令是否处理了 pipefail
- 交互提示是否有默认值
- 错误是否有明确的提示信息
- 是否需要同步更新 README
- 是否有官方提供的现成方案可以复用
